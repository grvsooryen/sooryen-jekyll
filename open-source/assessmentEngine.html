About Assessment Engine

Assessment engine is designed to assess mastery of student learning outcomes. Formative and summative assessments are designed and administered by instructors to be taken by students.

For more information, click here to visit bitbucket repository

Below mentioned Request Headers will be required in all API calls.

Cache-Control: "no-cache"
X-API-Key: Provided by the Assessment tool provider

Create Quiz

  • Create a new quiz for context.
  • Controller Name : Api::V1::QuizzesController#create
  • Endpoint : http://{root_path}/api/v1/quizzes
  • Method : POST
  • Required parameters :
Parameter Type Description
first_name String (Required) First name of user
last name String (Required) Last name of user
email_id String (Required) Email of user
user_id String (Required) User id provided by application which is using api
role String (Required) Instructor/ Student (but for this api only Instructor is allowed.)
quiz[:title] String (Required) Name of Quiz
quiz[:description] text (Required) Description / Instruction for Quiz.
quiz[:resource_link_id] String (Required) It should be the unique key to recognize the quiz
quiz[:quiz_type] String (Required) Type of quiz  [graded_assessment, survey] In v1 only graded_assessment is supported.
quiz[:shuffle_answers] boolean It will shuffle answer options
quiz[:time_limit_check] boolean It will check time_limit of quiz is present or not.
quiz[:time_limit] integer Value of time_limit in minutes.
quiz[:allow_multiple_attempt_check] boolean True/false value will check for allow_multiple_attempt_check.
quiz[:allowed_attempts] integer How many attempts are allowed for quiz value in integer here.
quiz[:score_filter] string Select category from highest  / lowest for list score of quiz.
quiz[:one_question_at_a_time] boolean It will check one question will be show at time or not.
quiz[:lock_question_after_answer] boolean It will lock question after answer at “true” value.
quiz[:show_quiz_response] boolean At “true” it will show quiz response
quiz[:once_after_each_attempt] boolean It will show quiz resource at once only at “true”
quiz[:show_correct_answers] boolean At “true” value it will show correct answer of question
quiz[:show_correct_answers_at_date] date It will show date of answer
quiz[:show_correct_answers_at_time] time It will show time of answer
quiz[:hide_correct_answers_at_date] date It will hide date of answer
quiz[:hide_correct_answers_at_time] time It will hide time of answer
quiz[:auto_publish] boolean
quiz[:unlock_at] datetime It will unlock the question after go on the next question.
quiz[:lock_at] datetime After next question previous question will be auto lock on true value
quiz[:context_id] integer It will give you course_id of quiz
quiz[:resource_link_id] string It will give you unique id of doc from where quiz is created.
Commit String “Next” value have to pass to go to next step.
role string Role of current_user

Returns Quiz object

Create Quiz submission

  • It will create attempt id for student and will send the attempt id to the tool consumer which will be needed by the tool consumer throughout the Attempt submission.
  • Controller Name : Api::V1::QuizSubmissionsController#create
  • Endpoint : http://{root_path}/api/v1/quizzes/{quiz_id}/quiz_submissions
  • Method : POST
  • Required parameters :
Parameter Type Description
context_id String (Required) It will be the course_id / section_id.
user_id Integer Id of current_user provided by tool consumer
quiz_id String (Required) It will be the resource_link_id provided by tool consumer.
first_name string first_name of user.
last_name string last_name of user.
Email string Email of user.
role string Role of user.

The response will contain submission_id, submission_attempt_id, total_pages, questions and quiz information.

Now this information will tell the tool consumer that the quiz is all at once or one by one(because if the quiz is all at once then all questions will be sent to the consumer, otherwise single question will be sent).

According to this,

  • If quiz is all questions at once then call Update Quiz Attempt api directly.
  • If quiz is one by one then call One By One Quiz Attempt api and for last question call Update Quiz Attempt api.

One by One Quiz Attempt

  • Controller Name : Api::V1::QuizSubmissionsController#whole_next_previous
  • Endpoint : http://{root_path}/api/v1/quizzes/:quiz_id/quiz_submissions/whole_next_previous
  • Method : POST
  • Required parameters :
Parameter Type Description
user_id Integer Id of current_user
context_id String (Required) It will be the course_id / section_id.
quiz_id String (Required) It will be the resource_link_id provided by tool consumer.
params[:commit] String “Next” or “Previous”

Pass params[:commit] as “Next” for next question and pass “Previous” for visiting the previous question. When last question appears then call Update Quiz Attempt api.
